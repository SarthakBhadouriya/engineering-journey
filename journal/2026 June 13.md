# Learning Journal Entry



###### **Date:** 13 June 2026



###### **Topic:** Running JavaScript Files with Node.js and Debugging File Path Errors



## Objective



###### Run a JavaScript file using Node.js from the VS Code terminal.



\---



## Initial Attempt



###### Command executed:



```powershell

node script01.js

```



###### Error received:



```text

Error: Cannot find module 'D:\\STUDY\\SWE\\Projects\\eng\_journey\_github\\engineering-journey\\script01.js'

```



\---



## Investigation Process



##### Step 1: Verify Current Working Directory



Executed:



```powershell

pwd

```



Result:



```text

D:\\STUDY\\SWE\\Projects\\eng\_journey\_github\\engineering-journey

```



###### Conclusion:

###### 

###### The terminal was operating from the expected project root directory.



\---



##### Step 2: Verify File Existence



Attempted:



```powershell

dir \*.js

```



###### Result:

###### 

###### No JavaScript files were found in the current directory.



###### Conclusion:

###### 

###### The problem was not with Node.js itself. The requested file did not exist in the current working directory.

###### 

\---



##### Step 3: Locate the Actual Script File



###### Examined the project structure and discovered the script was located in:



```text

JavaScript\\Exercises\\basic-js\\script.js

```



###### Additionally, the file had been renamed from:



```text

script01.js

```



to:



```text

script.js

```



\---



##### Step 4: Navigate to the Correct Directory



Executed:



```powershell

cd JavaScript\\Exercises\\basic-js

```



Then:



```powershell

node script.js

```



\---



## Result



###### The script executed successfully and produced the expected output:



```text

Hello, World!



8



\['apple', 'banana', 'cherry']



apple

banana

cherry



{ name: 'John', age: 30, city: 'New York' }

```



\---



## Root Cause



Node.js was searching for:



```text

engineering-journey\\script01.js

```



###### However:

###### 

###### 1\. The file was named `script.js`, not `script01.js`.

###### 2\. The file was located inside a nested subdirectory.

###### 3\. The current working directory did not contain the target file.



\---



## Things I Learned During This Experiment



##### 1\. Node Executes Relative to the Current Working Directory



When running:



```powershell

node filename.js

```



###### Node looks for the file relative to the directory shown by:



```powershell

pwd

```



###### If the file is not located there, Node cannot find it.



\---



##### 2\. The Current Working Directory Matters



###### Before running a file, I should verify:



```powershell

pwd

```



###### and ask:

###### 

###### > "Am I currently inside the folder that contains the file I want to run?"



\---



##### 3\. Terminal Errors Often Contain the Answer



###### The error message explicitly showed the exact path Node was trying to access.

###### 

###### Reading the full error message carefully can often reveal the problem immediately.



\---



##### 4\. Verify Assumptions Systematically



###### A useful debugging workflow:

###### 

###### 1\. Read the error.

###### 2\. Check the current directory.

###### 3\. Confirm the file exists.

###### 4\. Verify the filename.

###### 5\. Verify the path.

###### 6\. Re-run the command.

###### 

###### This approach converts assumptions into verified facts.



\---



## \## Key Takeaway



###### A "Cannot find module" error often has nothing to do with JavaScript code itself. Most commonly, it means Node.js cannot locate the target file because the filename, path, or current working directory is incorrect.
